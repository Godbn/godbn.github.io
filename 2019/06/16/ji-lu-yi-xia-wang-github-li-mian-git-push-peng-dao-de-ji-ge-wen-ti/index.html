记录往Github `git push` 几个问题

本文最后更新于：2020年6月21日 下午

引言

本来是想把做的一些前端项目上传到 github 上面进行储存一份但是之前没建立过本地仓库和git仓库之间的使用，引发下面问题：

一、fatal: Not a git repository

在使用 git add . 的时候出现了这样的错误：

fatal: Not a git repository (or any of the parent directories): .git

提示说没用.git这样的一个目录，我明明是从git上使用git clone进行下载的，后来才想明白是进错目录了。

不过如果碰到了上面的问题，解决方案：

二、error: failed to push some refs to

在使用 git push -u origin master 的时候出错：

原因是github中的README.md文件不在本地代码仓库中

使用下面命令进行代码合并 [:pull=fetch+merge]

这样本地仓库就多出了README.md文件

再执行 git push -u origin master 就可以完成上传了

三、Everything up-to-date

执行git push出现”Everything up-to-date”

原因呢：

  1. 没有 git add .
  2. 没有 git commit -m "提交信息"

如果上面两个步骤都成功执行，还出现这个错误是因为创建的目录下是空的，目录下必须有文件才能git push上传成功

在github上创建文件的时候，在新文件名后加/符号就是文件夹，但是这种方式只支持英文名目录，中文名目录不支持

要是想创建中文名文件夹，就通过客户端工具或终端命令行实现

如果在github一个文件夹下只有一个文件，那么删除这个文件的同时，它所在的文件夹也一同删除了
